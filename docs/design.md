# "AI 속에 숨어라" — VR 라스트맨 스탠딩 설계 문서

## 1. 게임 개요

AI NPC들 사이에 숨어서 AI인 척 행동하며, 다른 플레이어(사람)를 찾아서 제거하는 1인칭 VR 서바이벌 게임. 마지막까지 살아남으면 승리.

- **플랫폼:** Meta Quest 2/3 스탠드얼론
- **시점:** 1인칭 VR
- **플레이어 수:** 8~12명 + AI NPC 20~30명
- **판 길이:** 3~5분
- **아트 스타일:** 로우폴리 카툰
- **네트워크:** Photon PUN2
- **개발 기간:** 1주일
- **팀:** 4명 (맵 1, Unity 개발 2, AI 1)

---

## 2. 게임 플로우

```
[로비] → [매칭 대기] → [게임 시작] → [라운드 진행] → [결과 화면] → [로비]
```

### 로비
- 방 생성 / 참가 (PUN2 Room)
- 최소 4명, 최대 12명
- 호스트(방장)가 시작 버튼

### 게임 시작
- 모든 플레이어가 동일 외형으로 맵에 스폰
- AI NPC 20~30명도 동일 외형으로 맵 곳곳에 스폰
- 습득 칼 5~8개 맵 곳곳에 랜덤 스폰

### 라운드 진행 (3~5분)
- 자유롭게 돌아다니며 관찰
- 수상한 대상을 기본 칼 또는 습득 칼로 제거
- AI 오킬 3회 → 본인 사망
- 실제 플레이어 킬 → 오킬 카운트 영향 없음

### 승리 조건
- 마지막 생존 플레이어 1명 = 승리
- 타이머 종료 시 생존자 전원 승리 (또는 킬 수 기준)

## 3. 킬 시스템 & 오킬 페널티

### 무기

| 무기 | 소지 조건 | 리치 | 사용 횟수 |
|------|----------|------|----------|
| 기본 칼 | 항상 소지 | 1.5m (짧음) | 3회 |
| 습득 칼 | 맵에서 줍기 | 2.5m (김) | 1회 (사용 후 소멸) |

- 공격 쿨타임: 2초
- 킬 모션: 0.5~1초 (이 동안 무방비 — 다른 사람에게 노출 위험)

### 킬 판정 (호스트에서 처리)

```
플레이어가 공격
→ 호스트(심판)에 킬 요청 전송
→ 호스트가 대상 확인:
   ├─ 대상이 플레이어 → 킬 성공, 오킬 카운트 변동 없음
   └─ 대상이 AI → 오킬 카운트 +1
       ├─ 3회 미만 → 경고 UI 표시 ("오킬 2/3")
       └─ 3회 도달 → 본인 사망
```

### 오킬 카운트 규칙
- 단방향 증가만 가능 (리셋/감소 없음)
- 플레이어를 정확히 킬해도 카운트는 줄어들지 않음
- 3회 누적 시 즉시 사망

### 사망 처리
- 죽은 캐릭터는 바닥에 쓰러짐 → 3초 후 사라짐
- 사망한 플레이어는 관전 모드 (다른 생존자 시점)

## 4. AI NPC 행동 설계

### 목표
AI가 사람처럼 보여야 한다 (그래야 플레이어가 숨을 수 있음).

### 행동 — 단순 FSM (유한 상태 머신)

```
[배회] ←→ [멈춤] ←→ [주변 둘러보기] ←→ [다른 NPC 근처로 이동]
```

- **배회:** 랜덤 웨이포인트를 향해 걷기 (속도 = 플레이어와 동일)
- **멈춤:** 랜덤하게 2~5초 서 있기
- **둘러보기:** 제자리에서 고개 좌우 회전
- **근처 이동:** 가까운 NPC/플레이어 방향으로 걸어감 (군중 형성)

### 핵심 원칙
- AI 이동 속도 = 플레이어 이동 속도 (같아야 구별 불가)
- 상태 전환 타이밍에 랜덤성 부여 (기계적으로 안 보이게)
- AI끼리 겹치지 않게 간단한 회피만 적용
- 경로탐색은 NavMesh 기본만 사용 (Quest 성능 고려)

### 호스트 최적화
- 플레이어 근처 AI만 풀 업데이트 (10m 이내)
- 먼 AI는 0.5초 간격으로 업데이트
- 물리 연산 없음 — NavMeshAgent 이동만

## 5. 맵 설계

### 야외 마을
- 규모: 80m × 80m
- 중앙 광장 + 주변 골목/건물 외벽/노점
- 시야를 막는 구조물 (벽, 수레, 나무 등) → 은밀한 킬 가능
- 열린 공간 + 좁은 골목 혼합 → 긴장감 변화

### 스폰 배치
- **플레이어:** 맵 가장자리에 분산 스폰 (서로 떨어진 위치)
- **AI NPC:** 맵 전체에 균등 분포
- **습득 칼:** 5~8개, 맵 곳곳에 랜덤 스폰, 라운드 중 리스폰 없음

### 맵 경계
- 자연 장애물 (울타리, 절벽 등)로 제한

## 6. 네트워크 구조

### Photon PUN2 — 호스트 권위 방식

**호스트 (방장 = 심판)**
- AI NPC 전체를 움직이게 하는 역할
- 공격이 맞았는지 판정
- 오킬 카운트 관리
- 라운드 시간/승리 판정

**클라이언트 (나머지 플레이어)**
- 내 캐릭터 움직이면 다른 사람 화면에도 반영됨
- 공격하면 호스트에게 "나 이 사람 공격했어" 요청
- 호스트가 결과 알려줌

**방장 이탈 대응:**
- PUN2 MasterClient 자동 전환 기능 사용
- 새 호스트가 AI 시뮬레이션 이어받기

### 네트워크 대안 (나중에 전환 가능)
- Photon Fusion: 최신 버전, 틱 기반 동기화
- Unity Netcode + Relay: 무과금, 자체 관리

### 아키텍처 대안 (나중에 전환 가능)
- 분산 시뮬레이션: AI 부하를 여러 클라이언트에 나눔
- 전용 서버: 별도 PC/클라우드가 모든 로직 처리
